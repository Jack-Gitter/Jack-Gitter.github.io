idempotent actors to mitigate one of those issues

Attempt #3

In memory state machine representation

Only have onEntry() perform adjustments to javascript objects, then have actor.subscribe() persist those. Issues being that when actor.subscribe() fires we are already in the next state. If we fail to persist the work done, then we shouldn't be in that state. Rollbacks become difficult

mitigates also the async actor problem, as nodejs is single threaded and actor would fire before handler returns
